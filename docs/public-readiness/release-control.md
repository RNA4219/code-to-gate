# Public Readiness Release Control

本ドキュメントは、code-to-gate のリリース統制手順を記録する。外部公開・継続統制水準の技術統制継続運用に必要なrelease approval、rollback、artifact retention、schema compatibility確認手順を定義。

---

## Release Approval Procedure

### Pre-Release Checklist

1. **Quality Gate**: `npm run release:public:quality` passes
2. **Audit Gate**: `npm run audit:deps` passes (high/critical = 0)
3. **Evidence Pack**: `npm run evidence:pack` generates artifacts
4. **CI Verification**: GitHub Actions PR workflow passes
5. **Documentation**: README, CHANGELOG updated

### Release Steps

```bash
# 1. Quality verification
npm run release:public:quality

# 2. Audit verification
npm run audit:deps

# 3. Generate evidence artifacts
npm run build
node ./dist/cli.js analyze . --emit all --out .qh
node ./dist/cli.js readiness . --from .qh --out .qh
node ./dist/cli.js export sarif --from .qh --out .qh/results.sarif
npm audit --json > .qh/audit.json

# 4. Package verification
npm pack --dry-run

# 5. Publish (requires approval)
npm publish --access public
```

### Approval Requirements

- Technical lead approval (PR sign-off)
- Security review checklist completed
- Evidence artifacts generated and verified

## Rollback Procedure

### Within 72 Hours (npm unpublish)

```bash
# Check package versions
npm view @quality-harness/code-to-gate versions

# Unpublish specific version
npm unpublish @quality-harness/code-to-gate@<version>

# Or unpublish entire package (72h window only)
npm unpublish @quality-harness/code-to-gate --force
```

### After 72 Hours (npm deprecate)

```bash
# Deprecate specific version
npm deprecate @quality-harness/code-to-gate@<version> "Critical bug found, use <fixed-version>"

# Deprecate all versions
npm deprecate @quality-harness/code-to-gate "Package deprecated, use alternative"
```

### Emergency Rollback

1. Identify affected version
2. Publish patched version immediately
3. Deprecate affected version
4. Update CHANGELOG with rollback reason
5. Generate new evidence artifacts

## Artifact Retention

### CI Workflow Retention

| Artifact | Retention | Storage |
|----------|-----------|---------|
| PR artifacts | 30 days | GitHub Actions |
| Release evidence | 90 days | GitHub Actions |
| Coverage reports | 90 days | GitHub Actions |
| SARIF | 90 days | GitHub Code Scanning |

### Local Artifact Paths

| Artifact | Path | Cleanup |
|----------|------|---------|
| .qh/ | `.qh/` | Manual or git clean |
| coverage/ | `coverage/` | git clean -fdX |
| dist/ | `dist/` | npm run build regenerates |

### Cleanup Command

```bash
# Clean generated artifacts
git clean -fdX

# Or manual cleanup
rm -rf .qh coverage dist node_modules/.cache
```

## Schema Compatibility (ctg/v1)

### Contract

- **Schema version**: `ctg/v1`
- **Contract type**: Stable, backward compatible
- **Breaking changes**: Require new major version (ctg/v2)

### Compatibility Check

```bash
# Validate schema
node ./dist/cli.js schema validate .qh/findings.json
node ./dist/cli.js schema validate .qh/release-readiness.json

# Check for breaking changes (CI step)
git diff --name-only origin/main HEAD -- schemas/*.schema.json
```

### Migration Support

If schema changes required:
1. Document breaking changes
2. Provide migration guide
3. Update ctg/v1 → ctg/v2 version
4. Maintain backward compatibility layer

## Release Evidence Artifacts

### Required Artifacts

| Artifact | Path | Generated By |
|----------|------|--------------|
| release-readiness.json | `.qh/release-readiness.json` | `code-to-gate readiness` |
| findings.json | `.qh/findings.json` | `code-to-gate analyze` |
| audit.json | `.qh/audit.json` | `npm audit --json` |
| results.sarif | `.qh/results.sarif` | `code-to-gate export sarif` |
| coverage-summary.json | `coverage/coverage-summary.json` | `vitest coverage` |

### Evidence Verification

```bash
# Verify all artifacts exist
ls -la .qh/*.json .qh/*.sarif coverage/coverage-summary.json

# Validate schemas
node ./dist/cli.js schema validate .qh/findings.json
node ./dist/cli.js schema validate .qh/release-readiness.json
```
