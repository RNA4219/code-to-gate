# code-to-gate v0.1 Implementation Orchestration

## 1. 目的

v0.1 Local Release Readiness MVP を Shipyard-cp 型 `plan -> dev -> acceptance -> integrate` で実装する。

正本:
- `docs/requirements.md`: 製品定義、スコープ、機能要件
- `docs/acceptance-v0.1.md`: 完成条件、必須コマンド
- `docs/implementation-plan-v0.1.md`: Dev Slice 分解

## 2. Phase 定義

| Phase | 目的 | 完了条件 |
|---|---|---|
| Phase 0: Foundation | schema validation、fixtures、repo walker | acceptance input が揃う |
| Phase 1: Core Engine | TS/JS adapter、NormalizedRepoGraph、core rules | deterministic findings が出る |
| Phase 2: LLM Layer | structured output、unsupported claims isolation | risk/invariant/test seed が生成される |
| Phase 3: Gate & Export | readiness policy、downstream exporters | 4 integration schema validation 通る |
